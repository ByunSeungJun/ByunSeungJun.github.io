Fine Ludens는 데이터와 Vision AI로 생활 스포츠를 더 의미 있는 경험으로 재설계하는 스포츠테크 스타트업입니다. 핵심 서비스 Fine Play는 경기 영상을 올리면 하이라이트 선별·퍼포먼스 분석·리포트를 한 흐름으로 제공합니다. 저는 이 서비스를 0부터 만들어 iOS·Android·Web으로 출시·운영하며 백엔드·인프라·배포·결제 전반을 담당했습니다.
  • 공유 백엔드 아키텍처. Fine Play·Fine Day 두 서비스를 단일 Spring Boot와 통합 JWT 인증으로 설계하고 도메인 단위로 패키지를 분리해 운영 복잡도를 낮췄습니다.
  • 장애 근본해결. DB 커넥션 풀 고갈로 전 인스턴스가 재시작 루프에 빠진 장애를, HikariCP 풀 상한을 “인스턴스 수 × 상한 ≤ DB 한계”로 역산(4×10=40 / 151)해 150+ → 40~50개로 안정화했습니다.
  • 성능 최적화. 대시보드 N+1 제거(@ManyToOne 지연로딩)와 앱 리스트 N+1 감축(3N→N+1), 공용 이미지 캐시 파이프라인 ~45곳 적용으로 요청·메모리 비용을 줄였습니다.
  • 인증 · Redis. 리프레시 토큰 회전(RTR)·재사용 감지 시 일괄 무효화, setIfAbsent 분산락, 액세스 토큰 블랙리스트(TTL=잔여만료), Redis 장애 graceful degradation을 구현했습니다.
  • 인프라 · 배포 · 결제. AWS(EC2·ALB·RDS·ElastiCache) 기반 GitHub Actions CI/CD·Blue-Green 무중단 배포와, 토스 빌링키 정기/단건 결제·월구독 인프라를 구축했습니다.
  • 출시 · 콘텐츠 자동화. Flutter 앱을 App Store·Google Play에 출시(Apple 심사 반복 돌파)하고, 지표 시각화를 카드뉴스로 자동 생성하는 어드민 툴과 앱 대회 탭까지 연결해 접점을 넓혔습니다.
