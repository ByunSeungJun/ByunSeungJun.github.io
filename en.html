Fine Ludens is a sports-tech startup that redesigns grassroots sports into more meaningful experiences with data and Vision AI. Its core service Fine Play turns an uploaded match video into highlight selection, performance analysis, and a report in a single flow. I built this service from the ground up and shipped·operated it across iOS·Android·Web, owning backend, infrastructure, deployment, and payments end to end.
  • Shared backend architecture. Designed Fine Play and Fine Day on a single Spring Boot app with unified JWT authentication, splitting packages by domain to keep operational complexity low.
  • Root-cause incident fix. Resolved an outage where DB connection-pool exhaustion drove every instance into a restart loop — sizing the HikariCP cap by “instances × cap ≤ DB limit” (4×10=40 / 151) to bring it from 150+ down to a stable 40–50.
  • Performance optimization. Removed dashboard N+1 (@ManyToOne lazy loading), cut app-list N+1 (3N→N+1), and applied a shared image-cache pipeline in ~45 places, lowering request and memory cost.
  • Auth · Redis. Implemented refresh-token rotation (RTR) with bulk invalidation on reuse detection, setIfAbsent distributed locks, an access-token blacklist (TTL = remaining expiry), and graceful degradation on Redis failure.
  • Infra · Deploy · Payments. Built GitHub Actions CI/CD and Blue-Green zero-downtime deploys on AWS (EC2·ALB·RDS·ElastiCache), plus Toss billing-key recurring/one-time payments and monthly-subscription infrastructure.
  • Launch · Content automation. Shipped the Flutter app to the App Store·Google Play (clearing repeated Apple review), and widened touchpoints with an admin tool that auto-generates card-news from metric visualizations, wired through to the in-app tournament tab.
